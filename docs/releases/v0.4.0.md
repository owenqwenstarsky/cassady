## Cassady v0.4.0

Cassady v0.4.0 introduces a native desktop app preview alongside the existing terminal CLI, using the same agent engine, config, providers, tools, approvals, and JSONL conversation storage.

### Downloads

- macOS Apple Silicon: `cassady-v0.4.0-aarch64-apple-darwin.tar.gz`
- Linux x86_64: `cassady-v0.4.0-x86_64-unknown-linux-gnu.tar.gz`
- Linux ARM64: `cassady-v0.4.0-aarch64-unknown-linux-gnu.tar.gz`
- Windows x86_64: `cassady-v0.4.0-x86_64-pc-windows-gnu.zip`

The macOS and Linux archives contain `cass`, `cassady`, and `cassady-desktop`; launch the desktop app with `cass desktop`. The Windows archive contains `cass` and `cassady`. SHA-256 checksum files are included for every archive.

The desktop preview is distributed as a raw binary in the macOS/Linux archives, not as a DMG, app bundle, MSI, or Windows desktop executable. Windows users who want the desktop preview should use the Linux binary under WSL/WSLg for now.

### Highlights

- Adds `cass desktop`, which launches the `cassady-desktop` companion binary in the background so the window keeps running after the terminal closes.
- Adds a Tauri-based Cassady desktop app with a windowed chat surface, composer, transcript, streaming assistant text, reasoning display, tool blocks, approvals, cancellation, and model/access-mode controls.
- Reuses the real Cassady embedding API, config, provider registry, safety policy, built-in tools, and `~/.cass/conversations/*.jsonl` storage so desktop and CLI sessions stay interoperable.
- Ports the `cassady-web` terminal aesthetic into the app: IBM Plex fonts, dark chrome, scanlines, grain, vignette, glow effects, and shadcn-style primitives.
- Expands the embedding API with runtime model, access-mode, and reasoning-effort setters for embedded frontends.
- Adds `cassady-api` provider support for local API-style embedding flows without requiring an API key.

### Upgrade notes

No migration is required. The desktop app uses the existing Cassady config; run `cass setup` first if no provider is configured. Keep `cassady-desktop` next to `cass` or on `PATH` so `cass desktop` can find it. The desktop app preview is currently distributed for macOS Apple Silicon and Linux.

### Install from source

```sh
cargo install --git https://github.com/owenqwenstarsky/cassady --tag v0.4.0
```

### Verification

Built and tested with the GitHub Actions release workflow:

```sh
cargo test --locked --all-targets
cargo build -p cassady --release --locked --target aarch64-apple-darwin --bin cass --bin cassady
(cd cassady-desktop && cargo tauri build --target aarch64-apple-darwin --no-bundle -- --locked)
cargo build -p cassady --release --locked --target x86_64-unknown-linux-gnu --bin cass --bin cassady
(cd cassady-desktop && cargo tauri build --target x86_64-unknown-linux-gnu --no-bundle -- --locked)
cargo build -p cassady --release --locked --target aarch64-unknown-linux-gnu --bin cass --bin cassady
(cd cassady-desktop && cargo tauri build --target aarch64-unknown-linux-gnu --no-bundle -- --locked)
cargo zigbuild -p cassady --release --locked --target x86_64-pc-windows-gnu --bin cass --bin cassady
```
